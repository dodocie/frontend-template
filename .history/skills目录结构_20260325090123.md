根据《编码规则与要求V3(1).md》以及补充的四点核心原则，将这些规范拆分为以下 **5 个 Trae 核心 Skills (指令集/规则模块)**。

这种拆分方式遵循了“关注点分离”原则，方便 AI 在执行不同阶段任务时调用最相关的规则。

---

### Skill 1: 架构选型与环境规范 (Architectural Foundation)
**适用场景**：项目初始化、路由配置、新模块创建、环境排查。

*   **技术栈约束**：Node.js v20+，TS 严格模式。
*   **路由准则**：严禁手动模拟路由。Vue 使用 `vue-router`，React 使用 `react-router-dom`。
*   **UI 库选择**：
    *   Vue: PC (Element Plus) / Mobile (Vant).
    *   React: Mantine (优先) 或 Radix UI + Shancn/UI。
*   **环境上下文**：开发环境为 VS Code，需适配测试服务器部署链路。
*   **外链引用**：禁止 CSS `@import` 外链，必须本地化或使用非阻塞 `<link>`。

---

### Skill 2: 状态管理与逻辑驱动 (State & Logic Control)
**适用场景**：数据流设计、Store 编写、复杂业务流程实现。

*   **唯一真相原则 (SSOT)**：所有状态必须有明确的单一来源。
*   **状态工具**：Vue (Pinia) / React (Zustand/Jotai)。
*   **复杂流程控制**：涉及多步表单或复杂分支时，**强制使用 XState** 状态机，禁止堆砌 `if...else`。
*   **逻辑驱动优化 (针对 React)**：
    *   **事件驱动代替计算属性**：减少依赖链，避免过度响应式。
    *   **强制 Memo**：所有依赖 `state` 或 `props` 的计算属性必须用 `useMemo`；组件内方法必须用 `useCallback`。
*   **单一职责函数**：函数必须拆分为「数据处理」、「模型操作」、「文件操作」三大类。单个函数严禁超过 50 行。

---

### Skill 3: UI 开发与样式规范 (UI & Styling)
**适用场景**：组件编写、页面布局、主题定制。

*   **样式规范**：**强制使用 Tailwind CSS**。
*   **配置中心化**：Tailwind 变量及组件库主题必须在 `tailwind.config.js` 或全局配置文件中定义。
*   **声明式渲染**：严禁原生 DOM 操作。
*   **组件开发**：Vue 必须用 `script setup`；React 必须用 `Functional Components`。

---

### Skill 4: 数据处理与质量保障 (Data & Quality)
**适用场景**：接口对接、Mock 数据编写、日志调试、代码检查。

*   **数据中心化**：禁止组件内 fallback。Mock 数据必须在 `src/mocks/` 或 `src/api/mock/` 统一导出。
*   **关注点分离**：
    *   **数据处理函数**：仅负责转换/校验，禁止修改模型或读写文件。
    *   **模型操作函数**：仅负责 Store 读写。
    *   **文件操作函数**：仅负责 IO。
*   **日志规范**：禁止大量无意义打印。日志必须包含 `[模块名 + 函数名 + 上下文]`，且必须使用统一封装的日志工具。
*   **防御性编程**：必须显式处理 Loading 和 Error 状态。

---

### Skill 5: 工程化工作流与文档 (Workflow & Docs)
**适用场景**：需求变更、任务开始前、自动化任务、发布前。

*   **文档先于代码**：修改前必读 `docs/`，开发复杂功能前必须在 `docs/plans/` 建立 `.md` 计划。
*   **文档同步更新**：严禁使用 `v2/new` 等后缀，直接覆盖更新。
*   **脚本化思维**：重复性任务（清洗、构建、检查）必须编写 Node/Shell 脚本放入 `scripts/`，通过 `npm run` 调用。
*   **语义化规范**：变量/函数名高度语义化，复杂逻辑必须附带 TypeScript 类型或 JSDoc。

---

### 建议在 Trae 中的初始化配置 (`.trae/rules.json` 或类似配置)

你可以将上述 Skill 总结为一段 **System Prompt** 发给 Trae 的 AI 助理：

> **System Prompt:**
> "你是一个严格遵循项目规范的资深工程师。在开发过程中，请务必遵守以下核心准则：
> 1. **架构**：Vue3 (setup) / React (Hooks) + TS (Strict) + Tailwind CSS。
> 2. **逻辑**：遵循『关注点分离』和『单一职责』。React 中必须使用 `useMemo` 和 `useCallback` 优化性能。复杂流程必须用 XState。
> 3. **数据**：坚持『唯一真相』，Mock 数据必须中心化，严禁组件内硬编码。
> 4. **质量**：单个函数 < 50行，日志必须格式化，严禁原生 DOM 操作。
> 5. **流程**：开发前必看 `docs/`，复杂任务必写 `plans/`。
> 6. **环境**：适配 VS Code + 测试服环境，脚本放在 `scripts/`。
> 
> 在我开始提问前，请确认你已理解这些 Skill 的分类并会在后续代码生成中主动应用。"

### 针对你补充的第 4 点（测试服+VSCode）的分析：
由于涉及到测试服务器，建议在 `Skill 5` 中增加一条：
*   **远程同步感知**：在编写 `scripts/` 脚本时，需考虑代码同步到测试服的路径映射或环境变量切换，确保 Lint 检查和构建脚本在测试服务器环境下同样可用。
