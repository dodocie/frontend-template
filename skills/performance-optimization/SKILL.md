# Performance Optimization Skill

This skill enforces performance optimization patterns and best practices.

## 1. 代码分割与懒加载

*   **路由级别懒加载**: 所有路由组件必须使用懒加载。
*   **组件懒加载**: 非关键组件使用动态导入。

## 2. 图片与资源优化

*   **图片格式**: 优先使用 WebP、AVIF 等现代格式。
*   **响应式图片**: 使用 srcset 和 sizes 属性。

## 3. Bundle 分析与优化

*   **分析工具**: 必须使用 webpack-bundle-analyzer 或类似工具。
*   **第三方库**: 评估第三方库的体积影响。

## 4. 渲染优化

*   **虚拟滚动**: 长列表必须使用虚拟滚动。
*   **防抖节流**: 合理使用防抖和节流优化性能。
*   **组件按需加载**: 非首屏内容使用动态导入延迟加载。
*   **内存泄漏预防**: 在组件卸载时清理定时器、事件监听器等资源。

## 5. 性能监控

*   **核心指标**: 关注 Core Web Vitals（LCP、FID、CLS）等关键性能指标。
*   **性能测量**: 使用 Performance API 测量关键路径耗时。
*   **上报机制**: 建立性能数据收集和上报机制，持续监控应用性能。

## 6. 加载慢 vs 渲染慢的诊断与处理

> **AI 诊断前置问题**: 当用户反馈图片/视频加载或播放慢时，在排查之前应先向用户确认：
> **"请问是加载时间长（一直在转圈/白屏），还是已经显示/播放出来了但过程卡顿？"**
> 这决定了排查方向：前者重在网络与资源体积，后者重在渲染性能与硬件加速。

### 如何区分加载慢和渲染慢

*   **加载慢识别**:
    *   Network 面板显示资源下载时间长
    *   首次字节时间(TTFB)较长
    *   资源文件体积过大
    *   CDN 或服务器响应慢
    *   关键指标: FCP(首次内容绘制)和LCP(最大内容绘制)延迟

*   **渲染慢识别**:
    *   CPU 面板显示大量计算时间
    *   长时间的重排重绘
    *   复杂组件树更新
    *   频繁的状态变化导致重复渲染
    *   关键指标: FID(首次输入延迟)和TTI(可交互时间)延迟

### 加载慢的处理策略

*   **资源优化**:
    *   压缩图片、CSS、JS等静态资源
    *   使用现代图片格式(WebP/AVIF)
    *   实施资源预加载(preload/prefetch)，但需遵循预加载策略
*   **网络优化**:
    *   使用CDN加速资源访问
    *   启用HTTP/2和压缩传输
    *   实施缓存策略(强缓存/协商缓存)
*   **代码分割**:
    *   按路由和功能进行代码分割
    *   动态导入非关键组件

### 预加载策略（重要）

*   **数量限制**: 同时预加载请求不超过6-8个，避免超出浏览器并发连接限制
*   **优先级原则**: 仅预加载下一路由或用户高概率访问内容的关键资源
*   **资源类型限制**: 
    *   优先预加载CSS、关键JS、首屏图片等渲染阻塞性资源
    *   避免预加载大体积文件（如视频、音频等）
*   **时机控制**: 
    *   页面关键内容加载完成后(CUJ - Critical User Journey)再执行预加载
    *   避免在页面初始加载高峰期执行大量预加载请求
*   **设备感知**: 移动端应更谨慎使用预加载，考虑带宽和电池消耗
*   **监控指标**: 监控预加载成功率和对主页面加载的影响

### 渲染慢的处理策略

*   **组件优化**:
    *   使用React.memo/Vue.memo避免不必要的重渲染
    *   合理使用虚拟滚动处理长列表
    *   组件懒加载和按需渲染
*   **状态管理优化**:
    *   避免在渲染函数中进行复杂计算
    *   使用useMemo/useCallback缓存计算结果和函数引用
    *   合理拆分状态，避免大组件状态更新影响性能
*   **DOM操作优化**:
    *   避免频繁的DOM查询和修改
    *   批量处理DOM更新
*   **大型资源渲染优化**:
    *   图片：使用懒加载、响应式图片(srcset)、WebP/AVIF格式
    *   视频：使用流媒体协议(DASH/HLS)、适当的编解码器、按需加载
    *   设备硬件感知：检测设备性能并调整渲染质量(降级策略)

### 硬件性能感知与降级策略

*   **设备性能检测**:
    *   使用 Navigator API 检测设备硬件信息(CPU核心数、内存等)
    *   通过性能基准测试判断设备处理能力
    *   检测网络状况并调整资源加载策略
*   **动态降级机制**:
    *   根据设备性能自动调整渲染质量
    *   低端设备上禁用复杂动画和特效
    *   减少同时渲染的元素数量
    *   使用简化版组件或内容
*   **渐进增强**:
    *   基础功能优先保证可用性
    *   高级功能作为增强体验
    *   按照设备能力提供相应级别的体验

### 权衡与取舍策略

*   **优先级判断**:
    *   如果FCP时间过长，优先解决加载问题
    *   如果FCP正常但页面卡顿，优先解决渲染问题
*   **资源分配**:
    *   首屏加载时间 vs 交互响应速度的平衡
    *   预加载 vs 带宽占用的权衡
    *   缓存大小 vs 内存占用的权衡
*   **渐进式优化**:
    *   先解决影响最大的性能瓶颈
    *   逐步优化次要性能问题
    *   持续监控性能指标变化
